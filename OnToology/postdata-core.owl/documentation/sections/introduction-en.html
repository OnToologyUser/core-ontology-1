Introduction back to ToC

The postdata core ontology covers the aspects regarding poetic works and their manifestations.

We defne the classes: PoeticWork, Redaction and Ensemble.

We also incorporate classes that are not specific to poetry but represent a transversal knowledge and complete the relevant information.

With this aim, the following classes are therefore define:

  • Person and Organisation, to model the agents that intervene in the poetic work with different roles.
  • CreatorRole and Role, to model the authorship or other relations with the manifestation and transmission like editor or scribed, for example.
  • Place and Event, to represent origin places and mentioned events and places.

The core-ontology must be imported by the rest of ontologies in the network. For this reason, besides containing the mentioned classes, it also provides a set of common properties that have the same semantics in all the classes in which they are defined. In this way, it is possible to express semantics in an unambiguous way for properties that from this point of view conceptually represent the same.
