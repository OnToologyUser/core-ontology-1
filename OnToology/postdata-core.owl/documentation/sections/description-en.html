POSTDATA core ontology: Description back to ToC

As previously mentioned, the objective of this ontology is to represent the necessary knowledge to represent the characteristics related to poetic works and their manifestations.
It is important to note that some classes are alienated from the ontology edm

Therefore it has been modeled:

  • The class PoeticWork, abstract concept of an artistic creation.
    In POSTDATA these creations must be in verse (poem, play, song)
    For example: Os Lusíadas by Luis Vaz de Camões; La Commedia by Dante; the poem Ad te deus by an Anonymous author written in between the 7th and 9th centuries.
    This class is in turn alienated from the ontology edm as a class specialization ProvidedCHO and also a class specialization F1_Work.
  • La clase Redaction, covers representatives or manifestations of a work, that is, of an Poetic Work. Each version of a PoeticWork is its own redaction.
    For example: The palaeographic edition of A solis ortu usque ad occidua preserved in Lat. 1154 (BNF) and its critical edition are two Redaction of the same Poetic Work.
    This class is a specialization of the class F22_Self-Contained_Expression
  • The Ensamble class, is an abstract work that contains more than one Poetic Work. An Ensemble might be conceived as a collection of works, organized by author, by book song, by book of poems; or as a work mainly written in prose that contains more than one poetic pieces (each one of them being its own Poetic Work)
    Eg: Les fleurs du mal
    This class is a specialization of the class F17_Aggregation_Work

This ontology also models classes necessary to cover aspects necessary to deal with authorship, associated agents, places of origin and related events or dealt with in the redactions.

These classes have been aligned with other ontologies.

  • These are the Person class and the Organisation class, which are hierarchically dependent on the Agent class, as they share many functionalities. They have specialized from the class foaf:Person and from the class foaf:Organisation, respectively.

  • The Place class, specialization of edm:Place and geonames:Feature, represents a particular position, point, or area in space; a location.

  • The class Event, a specialization of edm:Event, represents a thing that happens or takes place and that can be mentioned in the Redaction of a PoeticWork.

  • It is important to indicate in the ontology core that there are two classes, the Role class and CreatorRole class, a specialization of the previous one.
    Role is used to represent the role of the agents related to PoeticWork, its manifestations, its transmissions and other related elements.
    CreatorRole is used for the function of a person or organization in the creation of a poetic work, redaction or other transmission element. This class also covers anonymous authorships and incorrect attributions. The following examples will show the use of this class, specifically to model a multiple authorship.

    • Also in the core there are properties of habitual use in different classes of the ontology because their semantics are the same for all of them. Many of these properties are defined through the reuse of some belonging to the vocabularies dc and dcterms, and to schema , for example, dc:title, schema:url or others like pdcore:commentary


    The core ontology will be imported by the rest of the ontologies of the POSTDATA ontology network.

    Use examples

    In order to facilitate the understanding of the use of some classes and properties of this vocabulary, some examples are provided below.

    Example 1. Authorship

    The example presented corresponds to the representation of the authorship part of the cantiga "Pedr' Amigo, quer' ora ũa ren' by Johan Baveca & Pedr' Friend of Sevilha", whose information is extracted from MedDB Lírica Profana Galego-Portuguesa database. Nettle Number 064022

    Example description

    The troubadours Johan Baveca & Pedr' Friend of Sevilha are the creators (pdcore:CreatorRole) of the cantiga that is modeled as a PoeticWork and its title is(PoeticWork1.title = “Pedr' Amigo, quer' ora ũa ren”).

    However, this type of composition was, allegedly, an improvisation similar to the freestyle battles in rap. This means that also each Stanza needs to be linked to the correspondent CreatorRole.

    This question is modelled in such a way that the CretorRole of the Cantiga pdcore:correspondsTo two persons:

    • Person1.name = “Johan Baveca”
    • Person2.name = “Pedr’ Amigo de Sevilha”

    The text presented is the Redaction: • PoeticWork1 – isRealisedThrough – Redaction1

    In regards to the editions, multiple references are cited. The most important one is the so-called “Seguida” (followed) which would be represented as:

    • Redaction1 -- retrievesTextLocation1

    We will show the RDF codes generated with the data of this example. It should be noted that the instances will be represented by their URIs created according to the following structure <http://postdata.linhd.uned.es/resource/{Class}/{UUID}>.
