POSTDATA core ontology: Description back to ToC

As mentioned in previously sections, the objective of this ontology is to provide the necessary knowledge for the poetic works and its manifestations representation.
It is important to note that some classes are alienated with the edm ontology.

Therefore following classes are modeled:

  • The pdcore:PoeticWork class, that represents an abstract concept of an artistic creation.
    In POSTDATA these creations must be in verse (poem, play, song)
    For example: Os Lusíadas by Luis Vaz de Camões; La Commedia by Dante; the poem Ad te deus by an Anonymous author written in between the 7th and 9th centuries.
    This class is in turn alienated with the edm ontology as a ProvidedCHO specialization and also with FRBRoo ontology as a F1_Work specialization.
  • The pdcore:Redaction class, that covers Poetic Work manifestations. Each version of a PoeticWork is its own redaction.
    For example: The palaeographic edition of A solis ortu usque ad occidua preserved in Lat. 1154 (BNF) and its critical edition are two Redaction of the same Poetic Work.
    This class is a specialization of the class F22_Self-Contained_Expression
  • The Ensamble class, that is an abstract work that contains more than one Poetic Work. An Ensemble might be conceived as a collection of works, organized by author, by book song, by book of poems; or as a work mainly written in prose that contains more than one poetic pieces (each one of them being its own Poetic Work)
    Eg: Les fleurs du mal
    This class is a F17_Aggregation_Work specialization.

This ontology also models necessary classes to cover aspects regards with authorship, associated agents, places of origin and related events.
These classes have been aligned with other ontologies. These are:

  • The Person class and the Organisation class, which are hierarchically dependent on the Agent class, as they share many functionalities. They are foaf:Person specialization and a foaf:Organisation specialization, respectively.

  • The Place class, specialization of edm:Place and geonames:Feature, that represents a particular position, point, or area in space; a location.

  • The Event class, a specialization of edm:Event, that represents a thing that happens or takes place and that can be mentioned in a PoeticWork Redaction.

  • In the ontology, there are also two important classes, the Role class and CreatorRole class, a Role specialization .
    Role is used to represent the role of the agents related with PoeticWork, its manifestations, its transmissions and with other elements.
    CreatorRole is used for represent the function of a person or organization in the creation of a poetic work, redaction or other transmission element. This class also covers anonymous authorships and incorrect attributions. The following examples will show the use of this class, to model a multiple authorship specifically .

    • Also in the core there are properties of habitual use in different classes of the ontology because their semantics are the same for all of them. Many of these properties are defined through the properties reuse that belong to the following vocabularies dc and dcterms, and to schema , for example, dc:title, schema:url or others like pdcore:commentary


    The core ontology must be imported by the rest of the ontologies of the network of ontologies.

    Use examples

    In order to facilitate the understanding of the some classes and properties of this vocabulary, some examples are provided below.

    Example 1. Authorship

    The example presented corresponds to the representation of the authorship part of the cantiga "Pedr' Amigo, quer' ora ũa ren' by Johan Baveca & Pedr' Friend of Sevilha", whose information is extracted from MedDB Lírica Profana Galego-Portuguesa database. Nettle Number 064022

    Example description

    The troubadours Johan Baveca & Pedr' Friend of Sevilha are the creators (pdcore:CreatorRole) of the cantiga that is modeled as a PoeticWork and its title is(PoeticWork1.title = “Pedr' Amigo, quer' ora ũa ren”).

    However, this type of composition was, allegedly, an improvisation similar to the freestyle battles in rap. This means that also each Stanza needs to be linked to the correspondent CreatorRole.

    This question is modelled in such a way that the CretorRole of the Cantiga pdcore:correspondsTo two persons:

    • Person1.name = “Johan Baveca”
    • Person2.name = “Pedr’ Amigo de Sevilha”

    The text presented is the Redaction: • PoeticWork1 – isRealisedThrough – Redaction1

    In regards to the editions, multiple references are cited. The most important one is the so-called “Seguida” (followed) which would be represented as:

    • Redaction1 -- retrievesTextLocation1

    We will show the RDF codes generated with the data of this example. It should be noted that the instances will be represented by their URIs (Uniform Resource Identifier) created according to the following structure <http://postdata.linhd.uned.es/resource/{Class}/{UUID}>.
    For example, the Person1 is identified by <http://postdata.linhd.uned.es/resource/Person/ccc591fe0-7f10-11e9-ac29-f2189886299d>
